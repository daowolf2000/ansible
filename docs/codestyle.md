Codestyle
---------

## Правила именование

- Все имена (переменных, плейбуков, задач) должны состоять из символов [a-z0-9_]
- Переменные которые создаются и используются только внутри задачи или плейбука должны начинаться с `x_`, а для 
  регистрирования выходных значений префикс `o_`. Это позволяет
    - снизить количество возможных коллизий и side эффектов
    - сократить имена переменных (нет необходимости писать слишком длинные имена) и повысить читаемость кода
- Имена возвращаемых значений должны начинаться с `get_`


## Tasks

- В задачах все необходимые переменные объявляем в начале файла, используя блок следующего вида:
  ```
    - name: Initialization variables
      ansible.builtin.set_fact: # noqa jinja[spacing]
        x_path      : "{{ path   | default('/usr/local/bin') }}"
        x_user      : "{{ user   | default(ansible_user_id) }}"
  ```


## Playbooks
- В плейбуках обязательно указываем 
  ```
  vars_files:
    - "{{ secret_vault }}/vars.yml"
  ```
  Это позволит при необходимости переопределить переменные используемые по умолчанию для проекта, а также загрузить необходимые ключи/пароли


## Прочее
- Для отладки в задачах и плейбуках использовать debug с verbosity = 1
